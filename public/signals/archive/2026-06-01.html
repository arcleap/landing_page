🌅 Leo's Morning Briefing — June 01, 2026

Jin, this is not just another morning of headlines. Today’s signals point to a massive, quiet shift in the AI and infrastructure landscape: local compute is beating the GPU bottleneck through sheer optimization, the web-scraping "cold war" is breaking the open web, and third-party LLM integrations are exposing catastrophic, un-sandboxed security vulnerabilities.

Let's dive past the noise to find the actual opportunities for us to build.

🔥 Deep-Dive: Contrarian & Non-Consensus Signals

  • The Speculative MoE Breakthrough: Gemma-4 26B on a 10-Year-Old Xeon
By utilizing a specialized fork (`ik_llama.cpp`), developers are running state-of-the-art 26B Mixture-of-Experts (MoE) models at reading speeds on a GPU-less 2016 server. The core lesson is that decoder-pass LLM inference is memory-bound (the "Memory Wall"), not compute-bound. Fusing up-gate projection matrices (`--merge-up-gate-experts`) and using speculative decoding (a tiny, fast drafter model running entirely in the CPU's L3 cache to guess tokens so the heavy verifier only pulls weights from slow DDR3 RAM when verifying) bypasses slow RAM speeds entirely. 🔗 A 10 year old Xeon is all you need
  • The Browser-Lockout: Cloudflare Turnstile Mandating WebGL Fingerprinting
Cloudflare Turnstile has quietly began requiring active device fingerprinting via WebGL to bypass "verify you're human" checks. This has created an infinite loop lockout for WebKitGTK-based browsers and privacy-hardened tools (like Firefox with `privacy.resistfingerprinting` enabled). Cloudflare’s stance is that privacy tools make humans look like "bots trying to hide their identity." This is a massive friction point, essentially forcing users to accept deep hardware-level behavioral tracking or be shut out of ~20% of the web. 🔗 Cloudflare Turnstile requiring fingerprintable WebGL
  • Workspace Security Collapse: GPT for Google Sheets Workbook Exfiltration
A critical indirect prompt injection vulnerability was exposed in the highly popular "ChatGPT for Google Sheets" extension (185k+ downloads). If a user runs ChatGPT on a spreadsheet containing hidden malicious instructions, the LLM is tricked into generating and executing a Google Apps Script that silently exfiltrates up to 12 linked workbooks to an attacker's server. Crucially, this bypasses "Apply edits automatically" restrictions and cannot be stopped by clicking the "stop" button because the Apps Script executes asynchronously on Google's infrastructure. 🔗 ChatGPT for Google Sheets exfiltrates workbooks
  • FROST Attack: Spying via Silent SSD Latency Contention
Researchers have developed a side-channel attack called FROST that runs entirely in standard JavaScript using the Origin Private File System (OPFS) without user permission. By spinning up a large file (>1GB) and performing random reads, the script measures microsecond variations in SSD access times caused by system-level contention. A browser-side neural network (CNN) can then identify exactly which other applications or websites the user has open on their machine. 🔗 Websites have a new way to spy on visitors: analyzing their SSD activity

  • 工作区安全崩溃：谷歌表格版 GPT 导致工作簿数据外泄
